## 3. Definitions

- **Seat** — one player agent: an isolated model session (own system prompt, history, provider call) behind the A2A boundary.
- **Referee** — the deterministic (non-model) owner of the board, legality, turn order, and game end; the only party a seat ever exchanges messages with (PRD-D4 topology).
- **Transcript** — the schema'd record of a match: every wire message plus each seat's *complete* private context.
- **Canary** — a per-seat unique token planted in a seat's private context whose *absence* from the other seat's world is mechanically checkable.
- **Concept/demo record** — a corpus record teaching one protocol concept (with a validated wire example) or one worked flow (with a replayable transcript).

## 4. Requirements

Normative per RFC 2119; each carries a PRD trace and acceptance criteria.

### 4.1 Grounding & pinning

**SPEC-R1 — Host-verification discipline.** Every upstream-protocol shape, name, or semantic this family relies on MUST have an HV row in §2; a build unit MUST NOT be dispatched while an HV row it depends on is unresolved; a resolution MUST carry a verbatim citation + date. *(→ PRD-G6)*
- **AC1** *Given* the B1 dispatch, *when* its units are enumerated, *then* every HV row named by SPEC-R3–R8 shows a resolution; an unresolved row blocks the dispatch (a review-gate check, recorded in the wave's handoff).

**SPEC-R2 — Version pinning.** Every artifact — type fixtures, corpus records, match transcripts, agent cards — MUST pin the resolved `protocolVersion` (PRD-D3); a consumer encountering a pin it does not support MUST fail with a coded failure, never silently proceed. *(→ PRD-G1, G6)*
- **AC1** *Given* a transcript or record pinning an unsupported version, *when* validated, *then* the verdict contains `A2A_PIN` and nothing downstream consumes the artifact.
- *Field-name reconcile note (HV-7 — RECONCILED at B0):* this SPEC's pin key `protocolVersion` and the card's `version` are **two distinct upstream fields**: HV-7's resolution confirms `AgentCard` requires BOTH `protocolVersion` (the protocol pin, upstream default `"0.3.0"`) and `version` (the agent's own version). Our artifact pin key matches upstream's `protocolVersion` exactly; card fixtures MUST carry both fields and MUST NOT conflate them.

### 4.2 Protocol core (PRD-G1)

**SPEC-R3 — Typed message model.** The core MUST represent A2A messages/parts per HV-4 as zero-dep typed structures, and MUST reject an unknown part kind or malformed message with a coded failure (not a throw). *(→ PRD-G1)*
- **AC1** *Given* the committed wire fixtures (transcribed from HV-4's resolution), *when* decoded then re-encoded, *then* they round-trip byte-identically.
- **AC2** *Given* a message with an unknown part discriminator, *when* validated, *then* the verdict contains `A2A_SCHEMA` at the offending path and the validator does not throw.

**SPEC-R4 — Task lifecycle state machine.** The core MUST implement the task states per HV-5 as an explicit transition table — the **full 9-state set** (`submitted · working · input-required · completed · canceled · failed · rejected · auth-required · unknown`, per HV-5's resolution); an illegal transition MUST be rejected with `A2A_STATE`; the four terminal states (`completed · canceled · rejected · failed`) MUST admit no exits. *(→ PRD-G1)*
- **AC1** *Given* the full (state × event) matrix over all 9 states, *when* the transition-table test runs, *then* every legal pair advances, every illegal pair yields `A2A_STATE`, and no terminal state has an outgoing edge.

**SPEC-R5 — Agent card.** The core MUST model + validate the AgentCard per HV-7; the dev tools MUST serve a card at the verified well-known path and MUST discover a peer by fetching + validating it. A card failing validation MUST NOT be used for discovery. *(→ PRD-G1)*
- **AC1** *Given* the arena's two seat cards + referee card, *when* fetched from the well-known path, *then* each validates; a fixture with a missing required field fails with `A2A_CARD` at its path.

**SPEC-R6 — Total validator.** ONE shared validator MUST judge any wire artifact (message, task, card, envelope): total (never throws — the `validateA2ui`/`validateRecord` posture), batch (reports every failure, not first-only), with the closed code set `A2A_SCHEMA · A2A_PIN · A2A_STATE · A2A_RPC · A2A_CARD`. Admission, gates, and pages MUST reuse it — no forks. *(→ PRD-G1; parity mirrors a2ui SPEC-N3)*
- **AC1** *Given* a malformed-input corpus (fuzzed + hand-built), *when* validated, *then* zero throws and each fixture's expected code set matches exactly.

**SPEC-R7 — RPC framing.** The core MUST frame/parse the JSON-RPC envelopes for the verified method surface (HV-3) incl. the error mapping (HV-9). Request/response correlation MUST be preserved; a malformed envelope yields `A2A_RPC`. Streaming methods (HV-6) are OUT of this wave unless B0 resolution shows the arena requires them. *Transport posture (per HV-2's resolution):* JSON-RPC is NOT "the baseline" — upstream defines **three equal-status core transports** (JSON-RPC 2.0 · gRPC · HTTP+JSON/REST) of which an agent MUST implement at least one and MAY choose JSON-RPC; this family scopes to JSON-RPC **as a deliberate choice** (compliant under the ≥ 1 floor; gRPC/REST out of scope per the PRD). *(→ PRD-G1)*
- **AC1** *Given* a request/response pair per verified method, *when* framed then parsed, *then* it round-trips with correlation intact; a malformed envelope yields `A2A_RPC`, not a throw.

**SPEC-R8 — Transports.** The core MUST expose a channel contract with two implementations: an **in-proc loopback** (first-class, deterministic, zero-network — what CI and the scripted arena use) and a **dev/server HTTP** transport (Node-scoped, never in a consumer bundle). Both MUST deliver messages in order without loss; switching transports MUST NOT change message semantics (transport-invariance). *(→ PRD-G1, G2)*
- **AC1** *Given* the same message sequence over loopback and HTTP, *when* both are decoded, *then* the sequences are identical and in order.

### 4.3 Arena (PRD-G2 — the flagship; refined by the tic-tac-toe LLD)

**SPEC-R9 — Deterministic referee owns the only shared truth.** The board, legality, turn order, and game end MUST be owned by deterministic code (never a model). Seats MUST NOT address each other; every message a seat receives MUST originate from the referee and MUST carry only the closed board-state shape (board cells, the seat's mark, last opponent *move*, legal moves, game status) — never any content authored by the other seat (no prose, no notes, no reasoning). *(→ PRD-G2; topology per PRD-D4)*
- **AC1** *Given* a completed match transcript, *when* the wire audit runs, *then* every seat-inbound message has referee origin and validates against the closed board-message schema; any extra field fails the audit.

**SPEC-R10 — Provable context isolation.** Each seat MUST run as a fully separate context: own system prompt, own conversation history, own provider call (sessions never shared, no common memory object). Each seat's private context MUST carry a unique canary token; the isolation gate MUST assert (a) seat A's canary appears nowhere in seat B's full context or in any wire message addressed to B, and vice versa, and (b) the wire audit of SPEC-R9 AC1. A committed **contaminated negative-control fixture MUST fail the gate** — proving the gate bites. *(→ PRD-G2; constraint C3)*
- **AC1** *Given* the flagship recorded match, *when* the isolation gate runs, *then* it exits 0; *given* the contaminated fixture, *then* it exits non-zero naming the leaked canary's location.

**SPEC-R11 — Turn-taking, illegal moves, and game end.** The referee MUST alternate turns; an illegal or malformed move MUST produce structured feedback to the offending seat (via the task lifecycle's input-required arm, HV-5) with a bounded retry (default 2); exhausting the bound MUST forfeit the game to the opponent; a per-move timeout MUST count as a malformed move. Game end (win/draw/forfeit) MUST notify both seats and complete the task(s). *(→ PRD-G2)*
- **AC1** *Given* a scripted seat emitting an illegal then a legal move, *when* the match runs, *then* the game completes with one structured-feedback exchange recorded; *given* a seat exceeding the bound, *then* the opponent wins by forfeit and the transcript records the reason.

**SPEC-R12 — Match transcript + deterministic CI backbone.** Every match MUST produce a schema'd transcript recording (a) every wire message in order and (b) each seat's complete private context (system prompt, every model message in/out, canary). A scripted-vs-scripted match MUST run under `npm test` — no network, no model — and be byte-stable across runs. The flagship fixture MUST be a recorded real-model match (provenance: models, date) committed to the repo. *(→ PRD-G2; constraint C4)*
- **AC1** *Given* two runs of the scripted match, *when* transcripts are compared, *then* they are byte-identical; *given* the committed flagship fixture, *then* the transcript-schema validator and the isolation gate both pass it.

**SPEC-R13 — Demo surface.** The site's match page MUST, in the static build, replay the committed fixture with zero network and zero keys; MUST present both seats' full contexts side-by-side beside the wire/board timeline; and MUST compute the isolation verdict **in-page from the transcript by running the same gate checker** (never a hardcoded badge). Live model-vs-model runs MUST be dev-only behind a proxy holding keys server-side (the a2ui ADR-0073 posture; `vite build` MUST carry no proxy path or key name). *(→ PRD-G2, G4)*
- **AC1** *Given* the static build, *when* the page loads, *then* the match replays and the verdict badge derives from the transcript; *given* a grep of `dist/` for the proxy mount/key names, *then* it is clean.

### 4.4 Corpus (PRD-G3; substrate per PRD-D5 — mirrored discipline, A2A-native schema)
