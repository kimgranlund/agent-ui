## Context

TKT-0008 asks for a CSS-native swiper family: `ui-swiper` hosting `ui-swiper-item` slides, an **infinite loop** the user can never bottom out, a customizable pagination/paddles/label chrome set with placement control, and first-class focus/wheel/selection UX, configured by `easing ¬∑ duration ¬∑ slides-in-view ¬∑ alignment ¬∑ orientation`. It is the designated live-fire shakedown of the TKT-0005 skill estate.

Three forces make this an ADR, not a bare ticket-to-build:

1. **The fleet has zero scroll-snap usage today.** A carousel's positioning is either native scroll-snap (CSS-native) or JS transform. Picking one is a fleet-first mechanism decision (F1) that every downstream test leg and gesture-handling choice keys off.
2. **"Infinite loop" is under-defined and the prior art does not actually implement it.** Kim's earlier CSS-native swiper (`/Users/kimba/Projects/adia/gen-ui-kit/.../swiper/swiper.class.js`) has a `loop` prop, but its `next()`/`prev()` at the boundary call `goTo(0)` / `goTo(lastPage)` ‚Äî a **rewind** (a visible snap back to the start), i.e. SwiperJS's `rewind`, **not** its `loop`. The ticket's "the user never hits a last slide; the wrap-around logistics are fully handled" and "provably seamless (no visible jump at the wrap seam)" cannot be met by promoting the prior art unchanged. The seam-jump DOM strategy is the design's hard center (F2), and it forces the loop's a11y model (position announcement when there is no "last" slide).
3. **Kim asked for `ui-swiper-pagination` / `ui-swiper-paddles` / `ui-swiper-label` as separate tags with placement control.** The prior art stamped that chrome as `[data-part]` children behind a `chrome` enum ‚Äî no placement freedom. Reconciling "separate tags the author positions" with "the swiper owns the dynamic dot count and wires the buttons" and "an agent can emit a usable carousel in one node" is a composition-model fork (F3, F5).

The novelty leg (predicted by the ticket) fires on the **mechanism**, not on geometry sizing: `geometry.md` already covers the shell (`pattern` size-class = "container + control-height rows", the named examples tabs/menu/dialog) and already names the carousel's paddles ("Nav icon in a standalone button ‚Äî carousel prev/next ‚Äî sized to match its context"). No new geometry row is invented; the greenfield is the scroll-snap viewport mechanism itself.

## Decision

Ship a **five-tag family** ‚Äî `ui-swiper` (coordinator, `extends UIContainerElement`, `tier: pattern`), `ui-swiper-item` (slide wrapper, `UIElement`, `tier: layout`), and the optional author-placed `ui-swiper-pagination` / `ui-swiper-paddles` / `ui-swiper-label` chrome (`UIElement`) ‚Äî realizing the SPEC/LLD contract this ADR ratifies, with these fork resolutions (all **firm recommendations**, none self-ratified):

- **F1 ‚Äî Positioning mechanism: scroll-snap (CSS-native).** The track is a `grid-auto-flow: column` (or `row` for vertical) scroll container with `scroll-snap-type: {axis} mandatory` and per-slide `scroll-snap-align: var(--ui-swiper-align)` + `scroll-snap-stop: always`; `grid-auto-columns` is computed from `--ui-swiper-columns` and the gap (the prior-art formula). Native scroll owns touch/trackpad gestures, momentum, and RTL for free. **This is the fleet's first scroll-snap surface** ‚Äî flagged for the reviewer.
- **F2 ‚Äî Infinite loop: clone-based scroll-teleport.** In `loop` mode the swiper clones the last `k` and first `k` real slides (`k = ceil(slides-in-view) + 1`, enough to fill the viewport past either seam), marks every clone `aria-hidden="true"` + `inert` (never focusable, never announced, never counted), starts scrolled to the first real slide, and ‚Äî on each scroll-snap settle that lands in a clone band ‚Äî teleports `scrollLeft`/`scrollTop` by exactly one real-set extent with `scroll-behavior: auto` (instant, sub-frame, invisible). Position is always reported as **real-index / real-count**; paddles never disable in loop mode.
- **F3 ‚Äî Composition: author-placed anchors the coordinator drives, with a boolean stamp fallback.** The chrome tags are **placement anchors** ‚Äî when the author places `<ui-swiper-pagination>` / `<ui-swiper-paddles>` / `<ui-swiper-label>` anywhere inside the swiper, the coordinator fills and wires them there (dynamic dot count, prev/next buttons, `aria-labelledby`). When no anchor is present and `[pagination]` / `[paddles]` is set, the swiper **stamps a default-placed anchor** ‚Äî pagination below the track, paddles overlaid inline-centered on the track (the LLD-C9/C10 default placement) ‚Äî so an agent (or a terse author) gets working chrome from one boolean. Present anchor always wins over the boolean. One drive path, one bootstrap.
- **F4 ‚Äî Selection surface: a bindable `active` prop committed by a `select` event.** `active` (reflected string: an item's `value`, or its real index as a string; `''` ‚áí first) is the active-slide identity; a user-driven change (scroll settle, paddle, dot, keyboard) commits it and emits `select { value, index }`; a programmatic write never echoes (binding hygiene). This is the ratified `ui-tabs` `selected`/`select` pattern (ADR-0019, renderer LLD-C8) applied to a carousel. No new event name; `change` is rejected (value-commit-flavored).
- **F5 ‚Äî Catalog posture: `ui-swiper` + `ui-swiper-item` are catalog-emittable; the three chrome tags are `EXCLUSION_ALLOWLIST`.** A content carousel (image gallery, card deck) is legitimate generative-UI content ‚Äî an agent emits `ui-swiper` with `[pagination]`/`[paddles]` booleans and `ui-swiper-item` children, no chrome tags needed (F3's stamp path). The chrome tags are author placement refinements, not content an agent composes; they take the ADR-0087 allowlist's second arm with reasoned entries (the Toast/ThemeProvider precedent).
- **F6 ‚Äî Autoplay: explicit NON-GOAL for v1.** Kim did not ask for it; auto-advance is a WCAG 2.2.2 (pause/stop/hide) liability and drags a timer + pause-on-hover/focus + a non-allowlist `autoplay-pause`/`autoplay-resume` event pair (all present in the prior art). Named foreseen extension, fenced.
- **F7 ‚Äî Pointer drag beyond native: deferred.** v1 is native scroll gestures only (touch/trackpad pan the track; mouse/keyboard use paddles, dots, the focusable track, and the native scrollbar). The prior art's mouse/pen click-drag is a promotable extension when asked, not v1.
- **F8 ‚Äî Wheel: native only, no axis translation.** The track captures wheel over itself natively (horizontal wheel/trackpad pans it). The swiper does **not** translate vertical wheel into horizontal advance (SwiperJS's `mousewheel` module) ‚Äî that hijacks page scroll and violates the one-owned-scroll-region law. Fenced.

The owning documents are `spec/swiper-family.spec.md` (contract) and `lld/swiper-family.lld.md` (realization); this ADR records *why*.

## Consequences

- **First scroll-snap surface in the fleet.** Its geometry and the loop seam are real-browser-only truths (jsdom has no scroll layout / no `IntersectionObserver` snap settle). The test plan carries an INSTRUMENT-BRIDGE browser leg for the seamless-teleport proof (scroll into a clone band ‚Üí assert post-teleport `scrollLeft` equals the real slide's snap target within Œµ AND the announced index is the real index); jsdom asserts only the DOM invariants (clone count, `aria-hidden`+`inert`, real-count math).
- **Clone-based loop duplicates DOM.** Clones must be inert and uncounted or they corrupt focus order, the pagination count, and AT announcements. The LLD makes clone lifecycle (create on connect / loop-toggle, rebuild on child mutation, destroy on disconnect) a first-class concern. Slides with `id`s duplicated into clones is a known hazard ‚Äî clones are deep clones with `id` stripped.
- **`internals.ariaRoleDescription` is load-bearing and must be browser-verified.** The region carries `aria-roledescription="carousel"` and each slide `"slide"` via `ElementInternals` (never a host attribute). jsdom's `ElementInternals` may not implement `ariaRoleDescription`; the build verifies it against real engines and reads it off `internals` in probes (the tabs `aria-selected`-via-internals precedent). If an engine lacks it, that is a build-time escalation, not a silent drop.
- **Five descriptors, five probe sets, doc + demo pages, barrels/exports/size.** A big family; the per-control bar (whole-shape, cross-engine, descriptor trip-wires, built-output) applies to each element. `slides-in-view=''` responsive-auto depends on production container-query CSS ‚Üí a built-output leg is required.
- **`stale ‚Üí re-verify`:** the a2ui default catalog coverage gate (`catalog/default/index.test.ts`) regenerates ‚Äî two new catalog rows (`Swiper`, `SwiperItem`) + three allowlist entries, allowlist residue must drain to zero. The whole-fleet descriptor/site coverage gates pick up five new descriptors.
- **No token edits.** The family consumes existing `--md-sys-color-*` roles only (idle dot `outline`, active dot `primary`/`on-surface`; paddles reuse `ui-button` ghost); the active-dot non-color signifier is **size** (ADR-0057). `tokens.css` is untouched by construction.
- **`duration`/`easing` shape programmatic advances only.** F1's scroll-snap positioning means native touch/trackpad gesture snaps use the UA's own smooth-scroll timing, which honors neither a custom `duration` nor a custom `easing` ‚Äî those tokens apply only to programmatic advances (paddle/dot/keyboard/`goTo`), and to honor a custom curve those must be driven by a JS scroll animation, **not** CSS `scroll-behavior: smooth` (which ignores author custom properties). The LLD (¬ß4/¬ß7) owns that mechanism; it is a real, accepted limit of choosing native scroll over transform, not a defect.
- **Reduced motion is a settled requirement (SPEC-R8), not an open fork:** programmatic goto collapses to instant and the loop teleport is already instant; no content is hidden. Recorded here so the ticket's named reduced-motion concern is visibly closed, not silently dropped.
- **The family-root token-consumption rule (ratified 2026-07-10, a build-time wall repaired, not a local deviation).** The LLD's original build named ONE shared `swiper.css` for all five descriptors ‚Äî the first family to give N>1 descriptors their own `{name}.md` while sharing one stylesheet (`ui-tab`/`ui-tab-panel` share `tabs.css` too, but neither has a descriptor of its own, so `family-coherence.test.ts`'s A4 (`{name}.css` must exist per descriptor) and invariant B (a `{name}.css` may declare/consume only its own `--ui-{name}-*` ‚à™ the shared allowlist) never applied to them). Building to that one-sheet design against the as-shipped gate produced a real wall: `swiper-item.css` etc. did not exist (A4), and the moment they were added, the leaf tags legitimately needing to *consume* the coordinator's `--ui-swiper-*` chain (e.g. `--ui-swiper-dot-size` in the pagination anchor) read as "reaching into another control's private token" ‚Äî exactly invariant B's target defect, but a false positive for a genuine one-family shared prefix. Resolved as a **coordinated repair, not a local deviation**: (1) the family now ships **five physical files** ‚Äî `swiper.css` stays the PRIMARY sheet (declares the whole family's `--ui-swiper-*` token table + the host's own anatomy), and `swiper-item/-pagination/-paddles/-label.css` each hold their own tag's anatomy, declaring **nothing** of their own; (2) `family-coherence.test.ts`'s invariant B gained a narrow **family-root consumption** amendment: in a folder with N>1 descriptors, the descriptor whose name equals the folder (the "primary") is the only one that may *declare* the shared `--ui-{primary}-*` prefix, and every OTHER descriptor in that folder may *consume* (never declare) it as family-shared ‚Äî a THIRD control's prefix is still out-of-family and still fails (the amendment's own negative control proves this). Declaration ownership (A2/B's core invariant) is untouched; only the consumption side widens, and only within one folder. `radio`/`radio-group` and `toast`/`toast-region` are unaffected ‚Äî neither consumes the other's prefix today, so the widening is dormant for them. ADR-0081 itself is **not edited** ‚Äî it is accepted, and its invariant B's *intent* (no reach into an unrelated control's private token) is exactly what the amendment preserves; only the mechanical predicate gained a family-folder case it hadn't needed before this build.

## Alternatives considered

- **F1 transform-based positioning (translateX per slide)** ‚Äî rejected: reimplements momentum, touch pan, and RTL that the platform gives for free; fights native scroll; is the opposite of Kim's "CSS-native" ask. Scroll-snap is proven cross-engine by the prior art.
- **F2 rewind (the prior-art `loop`)** ‚Äî rejected: a visible snap-back at the boundary is exactly the seam-jump the ticket forbids; it does not meet "the user never hits a last slide."
- **F2 pure scroll-wrap without clones (teleport end‚Üístart directly)** ‚Äî rejected: with `slides-in-view > 1` or `align != start` the end and start viewports are not visually identical, so the teleport is visible; clones are what make the seam pixel-identical.
- **F3 prior-art `chrome` enum stamping fixed parts** ‚Äî rejected: no placement control, low customizability ‚Äî the two things Kim explicitly asked to improve. Kept only as the *fallback* stamp mechanism, not the primary model.
- **F3 slots/parts of `ui-swiper` (no sub-tags)** ‚Äî rejected: the ask names sub-tags; sub-tags give the author a real element to position and style, and give the agent a catalog boundary (emit vs exclude) a slot can't.
- **F4 `change` for slide change** ‚Äî rejected: `change` is value-commit-flavored across the fleet; an active-view index is a selection, and `ui-tabs` already ratified `select` for exactly "which item in a set is active" (ADR-0019). Reusing it keeps the renderer's LLD-C8 two-way bind uniform.
- **F5 cataloguing the chrome tags** ‚Äî rejected: `ui-swiper-paddles`/`-pagination` carry no agent-authored content (the swiper fills them); an agent emitting an empty marker node is noise. The boolean stamp path (F3) serves agents without a catalog row.
- **F6 shipping autoplay behind a flag** ‚Äî rejected for v1: unasked, and it drags an accessibility-hostile timer + two non-allowlist events. Deferred as a named extension so the door stays open.
