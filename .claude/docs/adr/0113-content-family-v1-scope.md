## Context

Three content gaps, each with recorded evidence, and no vocabulary to close any of them:

- **Code.** Agents emit commands, configs, and payloads constantly; the fleet renders them as proportional, whitespace-collapsed prose. The docs site already built the correct answer privately — `site/lib/code-block.ts` (mono, `white-space: pre`, `overflow-x: auto`, textContent-only safety), consumed by ≥6 pages (the a2ui-live canvas JSON/HTML panes, every doc page's fenced example). The pattern is proven and trapped: site-local, fleet-invisible, catalog-unreachable.
- **Hyperlink.** `grep href catalog.json` returns nothing; `ui-text`'s `as` enum stops at `blockquote`. An agent structurally cannot say "here is the source." Closing this opens the catalog's **first navigation-security surface** (model-generated hrefs), which is why the member is admitted here but designed in its own record, ADR-0114.
- **Disclosure.** Long agent reports render all-or-nothing; the precursor fleet's `disclosure` was never ported and nothing replaced it.

Two standing laws bound the space before taste enters. **(1) The zero-dependency pillar** rules out every highlighter and sanitizer library — and ADR-0107 already settled that *vendoring runtime code is a dependency in costume* (the Phosphor exemption covers inert **data** only). **(2) The CSS-less-consumer law (ADR-0102)**: the primary consumer composes from catalog props and has no CSS verb, so overflow behavior, fold mechanics, and link safety must be component-owned defaults, never page-CSS homework. A third fact shapes the disclosure call: the fleet's *no-native-form-elements* law exists because FACE + `ElementInternals` owns **form participation** — `<details>` participates in no form, and `ui-modal` already wraps native `<dialog>` (ADR-0017), so the law does not bind here and the wrap question is decided on mechanics.

## Decision

**We will admit a content family into the fleet — `ui-code` + the `ui-text` hyperlink capability + `ui-disclosure` — hand-rolled under the zero-dep pillar, entering the default catalog in the same wave they ship, with the feed partition paid at the same moment.** One decision — the v1 scope + contract directions — in seven clauses; SPEC/LLD own mechanisms at build (PRD-G1..G5 trace).

1. **The v1 member set** *(PRD-G1/G2/G3; fork F1)*: `ui-code` (block-level verbatim code display), the hyperlink capability (on `ui-text`, per ADR-0114 — deliberately NOT a new tag), and `ui-disclosure` (summary/body fold). Ruled out for v1, with the fence in PRD §3: syntax highlighting, copy-to-clipboard, inline code chips, soft-wrap/line-numbers/max-height knobs, link `target`/`download` knobs, exclusive-accordion grouping, fold animation, rich summary slots. Each fenced item is a **new intake or a named foreseen extension**, never a rider.
2. **`ui-code` contract** *(PRD-G1; fork F2)*: **Display-class** (text-bearing display; no control height, no `[scale]`/`[size]` rows), reading the **`--ui-mono`** typeface constant (`dimensions.css` minted it naming "code blocks" as the consumer) at body-class metrics, with **`overflow-x: auto` owned by the component** — the ADR-0102 law applied: a code block that wraps mid-token or blows out a feed bubble is a destructive no-uptake rendering, so scrolling is Lane A, the component's own identity. **Content model: host-as-content** (light-DOM text is the code; `white-space: pre` preserves it; the newlines live in the text nodes so copy-paste fidelity is free) — the ui-text/ADR-0006 lineage, and exactly what the catalog's bindable `code → textContent` lane needs (the `Text.text` clobber path writes `textContent`; with **no stamp and no observer**, ui-code is a zero-machinery leaf — simpler than ui-text, nothing to heal). **`language` is a reflected, free-string, INERT metadata prop at v1** (the site helper's `data-lang` forward-hook, promoted): no rendering effect, no enum (nothing for ADR-0098 to gate). **Highlighting is fenced entirely**: a tokenizer is runtime code (the ADR-0107 rejection verbatim); the escape hatches, named: (a) light-DOM freedom — an app author MAY inject pre-highlighted children (the component guarantees only plain text under bound writes, which clobber to a text node); (b) a future opt-in adapter package outside the zero-dep core — its own intake. **Copy-to-clipboard is fenced** (fork F4-adjacent, decided): interactivity changes the class — Display stays passive; a copy affordance is app-layer composition (`Row > [Code, Button]`) or a future intake. Tokens: `--ui-code-*` (pad, surface; radius rides the fleet `--ui-radius-base` referent) — density-invariant quantities, build business.
3. **The hyperlink is a `ui-text` capability, not a component** *(PRD-G2)*: `as` gains `"a"` + a reflected `href` prop; the stamp becomes a real `<a>` (native link semantics, keyboard, focus — free, the ADR-0078 cl.4 doctrine). **Admitted here; designed in ADR-0114** — the scheme allowlist, enforcement point, `rel`/`target` policy, and bindability are that record's clauses, because model-generated hrefs are a security surface that must not ride a family scope ADR as a sub-bullet.
4. **`ui-disclosure` wraps native `<details>/<summary>`** *(PRD-G3; fork F3)*: the host adopts its light-DOM children into a component-owned `<details>` (`<summary>` from the `summary` prop + the children as body) — the ADR-0078 cl.4 stamp/heal lineage (a childList observer restores the invariant under parser streaming and factory writes; ui-select's options-move precedent). Chosen on mechanics over a bespoke button+region: native `<details>` gives the toggle behavior, `<summary>`'s button semantics + expanded/collapsed announcement, **find-in-page auto-expand of hidden content**, and the `name` exclusive-accordion substrate — all platform-free; a bespoke build re-implements every one against internals-ARIA limits (internals decorate the HOST, not a child header) and gets no searchability story. The no-native-form-elements law does not bind (Context; the ADR-0017 `<dialog>` precedent). Styling costs are real and accepted: the `::marker` is replaced by the fleet chevron affordance (`= font`, the geometry.md taxonomy), and **fold animation stays out of v1** (`::details-content` interpolation is too fresh cross-engine). Contract: **`open` is a reflected boolean prop, prop-as-source-of-truth under the ADR-0101 always-announce law** — every actual transition (user click, data write, find-in-page auto-expand, a future `name` auto-close) settles the prop and fires `toggle`; **`summary` is a string prop** (bindable on the wire; a rich `slot=summary` child is the foreseen extension — a prop keeps the heal invariant "children = body" simple). Geometry: **Pattern-class** — `geometry.md`'s table already lists "accordion": the summary row takes control height, the shell rides the space scale.
5. **Catalog + teaching, same wave** *(PRD-G1/G3/G5)*: the SPEC-N2 fleet-derived gate forces catalog-or-allowlist the moment a descriptor lands, so the build wave lands **`Code`** (`code`: string, bindable → `textContent`; `language`: string, non-bindable; display-only — no ADR-0019 seam slot) and **`Disclosure`** (`summary`: string, bindable; `open`: boolean, **two-way** `value:{prop:'open',event:'toggle'}` — the Menu/Popover/Tooltip/Modal precedent, consuming its one seam slot, CORRECTLY: no other Disclosure prop carries a user-mutated round-trip (`summary` is authored, the body is content), so nothing competes for the slot; `children` → body) in the same wave as the controls, intra-wave seed-and-drain permitted, end-of-wave allowlist residue **none**. The `Text` row's `href` cell lands per ADR-0114. The wave also lands the **report exemplar seed** (Card + Text + Code + a source link + a Disclosure-folded detail) and §5.2 Notes guidance in the ADR-0087 Fork-A style: *Text for prose · Code for verbatim/preformatted output (never for emphasis) · Disclosure for progressive detail (never hide the primary answer or a required control) · links for sources and references.*
6. **Feed-partition dispositions** *(fork F4)* — the ADR-0097 partition is TOTAL and mechanically owed (its Amendment's own lesson): **`Code` → IN** — an ask may legitimately show the thing it asks about ("run this command?" is a confirm archetype whose subject IS code; Code is light inline content — no overlay, no pagination, no dashboard); **`Disclosure` → OUT** — folding hides ask content, the Tabs "hides half the ask" reasoning verbatim (an ask must be fully visible and operable inline). `Text.href` rides Text's existing IN membership — stated consciously in ADR-0114's consequences, not smuggled. Partition becomes 24 IN / 14 OUT.
7. **Placement + adjacent consequences**: ordinary `controls/{code,disclosure}/` folders — no new package (nothing vendored, no data mass; the ADR-0107 cl.7 reasoning applies a fortiori). The **site's `code-block.ts` replacement is a booked follow-up consumer wave** (PRD-G5), deliberately not this scope — the exemplar and §5.2 land first so the fleet component is proven before the site swaps. The **family size budget (26 KB, ADR-0107 Amendment) likely re-bases** — measured at the build wave (`npm run size`, the manual discipline), not guessed here.

### Forks for Kim (each with a firm recommendation; the recommendation is the default absent an objection)

- **F1 — the v1 member set.** *Recommend: all three* (code + hyperlink + disclosure). Live alternatives: code-only (cheapest, but leaves the hyperlink as the one capability an agent cannot approximate by ANY composition — code at least degrades to Text, a link degrades to a dead URL string); deferring disclosure (but report folding is the forcing ask, and `Disclosure > Code` is the composition that keeps long-code knobs out of `ui-code`).
- **F2 — ui-code's content model.** *Recommend: host-as-content + bindable `code → textContent`* (clause 2): one content lane shared with `Text.text`, zero heal machinery, copy-paste free. The alternative — a `code` string prop rendered into component-owned DOM — forks the fleet content model (ADR-0006), adds template/clobber mechanics for nothing, and closes the light-DOM highlighting escape hatch.
- **F3 — the disclosure vehicle.** *Recommend: wrap native `<details>/<summary>`* (clause 4). The bespoke button+region alternative buys total styling freedom at the cost of re-implementing toggle, keyboard, and expanded-state announcement against internals-ARIA limits, and forfeits find-in-page auto-expand + the future `name` substrate — the platform is strictly ahead here.
- **F4 — the feed dispositions.** *Recommend: Code IN, Disclosure OUT* (clause 6). The live alternative — Code OUT on the charts' "report content" reasoning — misreads the class: charts are dashboards; code is the *subject* of a confirm ask. Either answer is a one-line, gate-visible edit in `feed-catalog.ts`.

## Acceptance

This is an **intake** ADR — realized in two stages:

- **Intake (this change):** the sibling PRD exists with gates green (`harness_checks.py prd` exit 0); this record and ADR-0114 pass `adr_check.py` and are indexed; the forks carry firm recommendations awaiting Kim; doc-review is dispatched. No code changes.
- **Build wave (M1/M2, separately dispatched):** descriptors land WITH same-wave catalog rows — SPEC-N2 gate green, allowlist residue none; the feed-partition gate green with the two clause-6 dispositions (and its negative control still biting); `ui-code` renders a multi-line snippet un-wrapped, scrolling inside its own box in a narrow container (whole-shape browser legs, both engines); `ui-disclosure` settles prop+attribute+`toggle` on user click AND on data-model write (two-way proven), announces expanded/collapsed via the native summary, and find-in-page reveals folded content (Chromium leg; WebKit structural probe per the instrument-bridge precedent); the report exemplar validates 0-`CATALOG` through `validateA2ui` and ships with the §5.2 guidance prose; ADR-0114's own acceptance holds.

## Consequences

- **The catalog's expressive floor rises where agents actually live** — commands, sources, long detail — and the site gains a fleet component where it currently maintains a private one (the follow-up consumer closes that loop).
- **`ui-disclosure` inherits the stamp machinery's costs**: a childList observer per instance and an adoption invariant to gate (the ui-text precedent bounds both). `ui-code` deliberately carries **zero** machinery.
- **`language` ships inert** — an honest placeholder: models will emit it (markdown-fence habit), and v1 renders it without effect. The fence will be pushed ("just highlight JSON"); the PRD §3 reason — a tokenizer is runtime code under the zero-dep pillar — is the standing answer, and any highlighter is a new intake.
- **Native `<details>` means the platform owns fold behavior** — UA differences (marker rendering, transition support) are absorbed as styling work, not logic; the v1 no-animation ruling is the cost of cross-engine honesty.
- **A parser nicety is accepted**: unlike native `<pre>`, the HTML parser does not strip a leading newline inside `<ui-code>` — authoring guidance in the descriptor, not machinery.
- **The feed partition grows** (24 IN / 14 OUT) — Code-in-asks is new surface area for ask composition; bounded by the partition gate and the archetype guidance.
- **Size budget re-base anticipated** (clause 7) — measured at build, the ADR-0107 Amendment precedent.
- **Stale → re-verify at the build wave:** catalog `catalog.json`/`factories.ts`/`index.test.ts` · `feed-catalog.ts` + its gate · catalog SPEC §5.2 rows + Notes · corpus shelf + derived prompt · `examples` barrels/`allSeeds` · the site follow-up booking.

## Alternatives considered

- **Adopt/vendor a highlighter (highlight.js, Shiki, Prism) or build a hand-rolled tokenizer.** Rejected: all are runtime code — the zero-dep pillar's chart-library rejection verbatim (ADR-0107: vendored runtime code is "a dependency in costume"); a hand-rolled multi-language tokenizer is a scope explosion with a permanent correctness tail. `language` stays inert; escape hatches named in clause 2.
- **Do nothing — keep `Text` + the site helper.** Rejected: the site helper proves the need and cannot serve it — it is unreachable from the catalog, and `Text` structurally collapses whitespace and wraps mid-token. The workaround is the evidence, not the answer.
- **A `variant="code"` on ui-text instead of `ui-code`.** Rejected: code is not a type role — it needs `white-space: pre`, its own overflow contract, mono family, and block chrome; forcing that through the typescale repoint matrix pollutes every axis (`truncate` on code? `as="h2"` code?) for no reuse.
- **A new `ui-link` component instead of extending ui-text.** Rejected — recorded fully in ADR-0114; summary: a link is text wearing semantics, which is precisely the `as` axis's charter (ADR-0078); a second text-like leaf forks the typography matrix and the content model for one attribute.
- **Bespoke button+region disclosure.** Rejected (fork F3): re-implements what `<details>` gives free, against internals-ARIA limits, and loses find-in-page + the `name` substrate. The styling ceiling is the accepted price.
- **`Disclosure.open` one-way (display-only row).** Rejected: a fold the model cannot observe desyncs the data model on every user toggle — the exact phantom-state class ADR-0101 closed for overlays; the two-way mark is shipped machinery.
- **Copy-to-clipboard on ui-code at v1.** Rejected (clause 2): mints interactivity, focus order, and clipboard-permission surface into a passive Display leaf; composition already expresses it.
- **Exclusive accordion (`name`) at v1.** Rejected: no forcing evidence; the native substrate makes it a cheap foreseen extension whose prop-sync mechanics clause 4's announce law already covers.
